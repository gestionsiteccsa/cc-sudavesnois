### Non-conformités détectées

✅ **Aucune non-conformité détectée après correction.**

**Note :** Une anomalie mineure a été identifiée et corrigée en cours d'audit :
- **10.7 — Focus visible** : Le remplacement de `:focus` par `:focus-visible` supprimait l'indicateur visuel sur mobile. **Corrigé** par l'ajout d'un style `:active` sur `.lightbox-trigger-wrapper` pour préserver le feedback visuel sur les interactions tactiles.

### Critères conformes

**Thème 1 — Images**
- **1.1** ✅ Images porteuses d'information : Toutes les `<img>` ont un attribut `alt` descriptif (ex: "Portrait de Ali LAMRANI, membre titulaire du Conseil Communautaire")
- **1.2** ✅ Images décoratives : Les placeholders avec initiales ont `aria-hidden="true"` car l'information est déjà présente en texte adjacent
- **1.3** ✅ Alternatives pertinentes : Les `alt` décrivent la fonction et le contexte, pas juste "photo"

**Thème 3 — Couleurs**
- **3.1** ✅ Information pas uniquement par couleur : La distinction titulaire/suppléant utilise une bordure pointillée + texte "(suppléant)", pas seulement la couleur
- **3.2** ✅ Contraste texte/fond : Texte blanc sur fond bleu primary (#064d7e) = ratio ~8.5:1 (conforme AA)

**Thème 6 — Liens**
- **6.1** ✅ Liens explicites : La lightbox n'utilise pas de liens ambigus
- **6.2** ✅ Liens avec intitulé : Tous les éléments interactifs ont des labels ARIA

**Thème 7 — Scripts**
- **7.1** ✅ Composants accessibles : La lightbox utilise `role="dialog"`, `aria-modal="true"`, `aria-label`
- **7.2** ✅ Alternative au script : Le contenu reste accessible sans JS (images visibles, texte présent)
- **7.3** ✅ Contrôle clavier : Lightbox ouvrable via Entrée/Espace, fermable via Échap, Tab piégé dans la modale
- **7.4** ✅ Changement de contexte : La lightbox s'ouvre uniquement sur action explicite (clic ou touche)
- **7.5** ✅ Messages de statut : Non applicable (pas de message dynamique)

**Thème 8 — Éléments obligatoires**
- **8.3** ✅ Langue déclarée : `<html lang="fr">` présent dans base.html
- **8.5** ✅ Titre de page : "Conseil Communautaire | Communauté de Communes Sud-Avesnois"

**Thème 9 — Structuration**
- **9.1** ✅ Hiérarchie des titres : h1 (page) → h2 (section) → h3 (commune) → h4 (membre) — cohérent
- **9.2** ✅ Structure du document : Landmarks HTML5 présents (header, main, section, article)

**Thème 10 — Présentation**
- **10.1** ✅ Présentation par CSS : Aucune balise de présentation dépréciée
- **10.4** ✅ Zoom 200% : Layout flex-wrap s'adapte automatiquement
- **10.11** ✅ Reflow : Pas de scroll horizontal à 320px (layout vertical en mobile)

**Thème 12 — Navigation**
- **12.1** ✅ Accès multiples : Liens d'évitement présents dans base.html

**Thème 13 — Consultation**
- **13.1** ✅ Langue du document : `lang="fr"` sur `<html>`

---

### Recommandations

1. **Ajouter `:active` pour mobile** : Comme indiqué dans la NC 10.7, ajouter un style `:active` pour préserver le feedback visuel sur tactile
2. **Test utilisateur** : Vérifier la lightbox avec VoiceOver/TalkBack pour s'assurer que l'annonce "Aperçu de l'image" est suffisamment explicite
3. **Taille des images** : Pour de meilleures performances, envisager des versions réduites des photos pour la grille (thumbnail) et la version pleine taille pour la lightbox
