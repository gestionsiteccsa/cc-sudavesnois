#### Thème 7 : Scripts

**[NC] 7.1.1 — Script générant ou mettant à jour du contenu**
- **Niveau :** A
- **Élément :** Popup CTG (#ctg-popup-overlay) — modale générée statiquement mais affichée/cachée via JS
- **Problème :** La popup s'affiche via `overlay.style.display = 'flex'`. Le contenu interactif (boutons Fermer/Découvrir) est présent dans le DOM mais caché avec `display: none`. Les lecteurs d'écran ne devraient pas les annoncer. Cependant, le `role="dialog"` et `aria-modal="true"` sont corrects.
- **Statut :** ⚠️ À vérifier — test NVDA nécessaire pour confirmer.

**[NC] 7.1.2 — Script contrôlant un composant UI**
- **Niveau :** A
- **Élément :** Bouton "Fermer la popup" — `onclick` n'est pas utilisé (bon point), l'écouteur est en JS.
- **Problème :** Absence de gestion du **focus trap** dans la popup. Lorsque la popup est ouverte, le focus doit être piégé à l'intérieur.
- **Impact :** 🔴 Critique
- **Correction :** Ajouter un véritable focus trap (cf. code ci-dessous).

**Exemple de correction (focus trap) :**
```javascript
function trapFocus(container) {
    const focusable = container.querySelectorAll(
        'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
    );
    const first = focusable[0];
    const last = focusable[focusable.length - 1];
    
    container.addEventListener('keydown', function(e) {
        if (e.key !== 'Tab') return;
        if (e.shiftKey && document.activeElement === first) {
            e.preventDefault();
            last.focus();
        } else if (!e.shiftKey && document.activeElement === last) {
            e.preventDefault();
            first.focus();
        }
    });
}
```

#### Thème 13 : Consultation

**[NC] 13.1.1 — Contenu dynamique (popup)**
- **Niveau :** AA
- **Élément :** Popup CTG avec `aria-live="polite"` manquant
- **Description :** L'apparition de la popup n'est pas annoncée par un `aria-live`. Bien que le `role="dialog"` et `aria-modal="true"` soient présents, certains lecteurs d'écran ne détectent pas automatiquement l'ouverture de la popup sans `aria-live`.
- **Impact :** 🟠 Majeur
- **Correction :** Ajouter `aria-live="assertive"` sur la popup OU utiliser JavaScript pour gérer l'annonce :

```javascript
function showPopup() {
    overlay.style.display = 'flex';
    document.body.style.overflow = 'hidden';
    overlay.setAttribute('aria-live', 'assertive');
    closeBtn.focus();
}
```

#### Thème 8 : Éléments obligatoires

**[NC] 8.2.1 — HTML invalide (CRITIQUE)**
- **Niveau :** A
- **Élément :** `<svg>` dans le bouton "Suivant" désactivé de la pagination
- **Description :** Ligne 144 du template `journal.html` :
  ```html
  <svg class="h-5 aria-hidden="true" focusable="false"  w-5" xmlns="..." viewBox="0 0 20 20" fill="currentColor" aria-hidden="true">
  ```
  L'attribut `class` contient `aria-hidden="true"` au lieu que ce soit un attribut séparé. Le HTML est invalide et le SVG aura une classe `aria-hidden="true"` incorrecte. L'attribut `aria-hidden="true"` n'est *pas* appliqué au SVG car il est absorbé dans le `class`.
- **Impact :** 🔴 Critique — 1) HTML invalide ; 2) Le SVG n'est PAS masqué des lecteurs d'écran, il sera annoncé.
- **Correction :** 

```html
<!-- AVANT -->
<svg class="h-5 aria-hidden="true" focusable="false"  w-5" xmlns="..." viewBox="0 0 20 20" fill="currentColor" aria-hidden="true">

<!-- APRÈS -->
<svg class="h-5 w-5" xmlns="http://www.w3.org/2000/svg" viewBox="0 0 20 20" fill="currentColor" aria-hidden="true" focusable="false">
```

**[NC] 8.2.2 — Balises malformées (span > div)**
- **Niveau :** A
- **Élément :** Bloc "Indisponible" dans la carte journal
- **Description :** Un `<div>` est inclus à l'intérieur d'un `<span>`. Bien que techniquement toléré, c'est invalide HTML5. Cela peut causer une fermeture automatique de la balise `<span>` par le navigateur avant le `<div>`.
- **Impact :** 🟡 Mineur
- **Correction :** Remplacer le `<div>` interne par un `<span>`.

```html
<!-- AVANT -->
<span class="inline-flex items-center px-4 py-2 bg-gray-300 ...">
    <div class="bg-red-100 dark:bg-red-900 text-red-800 dark:text-red-200">
        <span>Indisponible</span>
    </div>
</span>

<!-- APRÈS -->
<span class="inline-flex items-center px-4 py-2 bg-gray-300 ...">
    <span class="bg-red-100 dark:bg-red-900 text-red-800 dark:text-red-200">
        <span>Indisponible</span>
    </span>
</span>
```
